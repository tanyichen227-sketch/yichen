# 当前版本前后端功能与操作说明

> 说明：本文档只基于当前前台页面已经能看到、能操作的功能编写，不把后台代码里已经预留但前台未展示的能力当作“已上线功能”来写。

## 1. 项目定位

当前系统是一个面向知识管理与智能问答的前后端分离平台，前台负责知识库管理、聊天问答、任务执行、文件管理、模型管理、广场分享和个人设置，后端负责认证、文档处理、RAG 检索、知识图谱、会话存储和页面所需的数据服务。

从用户视角看，这套系统最核心的体验是：

1. 先把资料放进知识库。
2. 再通过聊天或检索对知识库发问。
3. 需要更复杂的动作时，切换到任务模式。
4. 需要分享或协作时，进入知识广场和共享详情页。

## 2. 前端功能介绍

### 2.1 登录与注册页

登录页是系统入口，当前页面提供：

- 邮箱登录
- 邮箱注册
- 找回密码 / 重置密码
- 表单切换
- 登录成功后的页面跳转

页面视觉上带有动效背景和标题动画，主要作用是完成账号进入系统前的身份校验。

### 2.2 主界面布局

登录后进入主应用布局，左侧是侧边栏，右侧是功能页面。

侧边栏当前可见的主入口包括：

- 知识库
- AI 对话
- 任务模式
- 历史记录
- 文档创作
- 文件管理
- 模型管理
- 系统设置

底部还提供：

- 全局搜索
- 语言切换
- 用户信息菜单
- 退出登录

页面级的全局能力包括：

- `Ctrl + K` 打开全局搜索
- 右侧智能助理
- 回到顶部按钮
- 页面切换动效

### 2.3 知识库列表页

知识库页是系统的核心入口，当前可见功能包括：

- 搜索知识库
- 新建知识库
- 星标收藏
- 置顶排序
- 删除知识库
- 拖拽排序
- 查看最近访问

用户可以把常用知识库放到前面，也可以通过搜索快速定位目标知识库。

### 2.4 知识库详情页

知识库详情页是“上传资料、做检索、配知识图谱、做设置”的主页面。当前页面上能看到的功能比较完整，主要包括四块。

#### 2.4.1 数据集管理

页面上可见的操作有：

- 搜索文件
- 按状态筛选文件
- 上传文件
- 导入网页链接
- 删除单个文档
- 开关启用 / 禁用
- 分页浏览

上传时页面提示支持 `PDF`、`DOCX`、`TXT`，并显示单文件大小限制。上传完成后，文档会进入当前知识库的数据集列表。

#### 2.4.2 RAG 检索模块

当前前台已经提供两种检索方式切换：

- `LangChain`
- `原生实现`

同时，检索面板里还能看到以下配置项：

- 使用知识图谱
- 检索策略
- 返回文档数 `Top-K`
- 相似度阈值
- 向量权重
- BM25 权重
- Rerank 重排序

页面里也提供了“先向量化，再检索”的操作按钮，说明当前流程是先把文档处理成可检索状态，再进行问答。

#### 2.4.3 检索结果展示

当用户执行检索后，页面会显示：

- 最终回答
- 参考来源
- 详细处理过程

这意味着系统不是只给结果，还会告诉用户答案来自哪些文档，方便追溯和核对。

#### 2.4.4 知识图谱与设置

知识库详情页下方还包含知识图谱设置模块和知识库设置卡片，当前前台能看到的操作重点是：

- 保存知识库设置
- 删除知识库
- 进入知识图谱配置

这部分更适合在答辩或演示时解释为“知识库级别的高级配置入口”。

#### 2.4.5 笔记模块

知识库详情页里还有一个笔记区，可以：

- 新建笔记
- 切换笔记
- 删除笔记

它更像是知识库内部的随手记录区，适合保存临时结论或使用提示。

### 2.5 聊天页

聊天页是用户最常用的问答入口，当前可见功能包括：

- 新对话

## 3. 前端实际操作流程

### 3.1 第一次进入系统

1. 打开登录页。
2. 使用邮箱登录，或先注册账号。
3. 进入主界面后，先确认左侧侧边栏和右侧内容区正常显示。

### 3.2 新建知识库

1. 在左侧点击“知识库”。
2. 点击“新建知识库”或左侧“快速新建”。
3. 输入知识库名称并确认。
4. 创建完成后，回到知识库列表查看新卡片。

### 3.3 上传资料并建立知识库

1. 进入某个知识库详情页。
2. 在数据集区域点击“上传文件”。
3. 选择 `PDF`、`DOCX` 或 `TXT` 文件。
4. 上传完成后查看文档列表。
5. 如需补充网页内容，可使用“导入链接”。

### 3.4 做检索和问答

1. 在知识库详情页先完成向量化处理。
2. 在检索输入框里输入问题。
3. 选择 `LangChain` 或 `原生实现`。
4. 根据需要调节检索策略和参数。
5. 查看最终回答、来源和详细处理过程。

### 3.5 使用聊天页

1. 进入“AI 对话”。
2. 选择一个模型。
3. 如需知识库增强，打开知识库问答开关并选择知识库。
4. 输入问题后发送。
5. 查看流式输出、引用来源和思考过程。

### 3.6 使用任务模式

1. 进入“任务模式”。
2. 选择示例任务或直接输入任务描述。
3. 决定是否使用知识库、是否联网搜索。
4. 选择执行模型。
5. 按 `Ctrl + Enter` 或点击开始执行。
6. 观察步骤流转和最终结果。

### 3.7 管理文件和模型

1. 在“文件管理”里查看所有文档、预览、删除和置顶。
2. 在“模型管理”里查看本地模型、下载新模型、修改 Ollama 地址。

### 3.8 分享和协作

1. 打开“知识广场”。
2. 搜索或筛选目标知识库。
3. 进入共享详情页后查看文档、版本和评论。
4. 如有权限，可邀请协作者、上传文档或提交 PR。

### 3.9 个人与系统设置

1. 进入“系统设置”调整 API Key、数据源、模型、OCR、外观等选项。
2. 进入“用户中心”修改头像、昵称、简介和语言。
3. 完成后返回业务页面继续使用。

## 5. 后端运行与前后端联调

### 5.1 后端启动

后端启动后，通常可以访问：

- 服务根地址：`http://localhost:8000`
- 接口文档：`http://localhost:8000/docs`

常见启动方式是：

```powershell
cd RagBackend
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

### 5.2 前端启动

前端启动方式是：

```powershell
cd RagFrontend
npm install
npm run dev
```

我已经在本地跑过一次构建，当前前端可以正常打包通过。

### 5.3 联调注意事项

- 前端需要后端服务正常返回数据。
- 聊天和知识库功能依赖模型服务。
- 本地模型依赖 Ollama。
- 云端模型需要对应 API Key。
- 上传完文件后，最好先完成向量化或解析，再进行问答。

## 6. 使用注意事项

1. 这份文档写的是“当前前台实际可见功能”，不是把后台所有预留模块都算进去。
2. 如果某些设置项在界面上只是入口或占位，说明它们更适合描述成“配置页”而不是“完整业务页”。
3. 聊天页和任务页都支持流式输出，等待时间会比普通表单提交稍长。
4. 知识库详情页里，只有解析完成的文档才更适合参与检索。
5. 如果模型管理页里看不到可用模型，通常先检查 Ollama 服务是否已经启动。
6. 知识广场和共享详情页更偏协作展示，适合在客户介绍里强调“分享、订阅、协作、版本管理”。

## 7. 详细功能说明

这一部分把“功能是什么、怎么操作、用了什么技术”拆开写，方便你直接拿去给客户介绍或者做答辩讲稿。

### 7.1 登录、注册与密码重置

**功能是什么**

- 用邮箱注册新账号。
- 使用邮箱和密码登录。
- 忘记密码后通过邮箱验证码重置。

**怎么操作**

1. 在登录页切换到注册页并填写邮箱和密码。
2. 注册成功后返回登录页。
3. 登录后系统会自动记录登录状态。
4. 如果忘记密码，点击“忘记密码”进入重置流程。
5. 按提示输入邮箱、验证码和新密码，完成重置。

**用到的技术**

- 前端表单交互由 Vue 组件状态控制。
- 登录态由 JWT 管理，前端路由守卫会检查是否已登录。
- 密码重置依赖邮箱验证码流程，后端负责验证码生成与校验。
- 页面加载和跳转使用 Vue Router。

### 7.2 知识库列表与管理

**功能是什么**

- 创建知识库。
- 搜索知识库。
- 星标收藏和置顶。
- 拖拽排序。
- 删除知识库。
- 查看最近访问。

**怎么操作**

1. 点击“新建知识库”输入名称。
2. 创建完成后在列表中查看新卡片。
3. 使用搜索框快速定位知识库。
4. 通过星标和排序把常用知识库放前面。
5. 需要删除时点击卡片删除按钮并确认。

**用到的技术**

- 知识库列表采用卡片式布局，提升浏览效率。
- 搜索为前端实时筛选。
- 星标、置顶、最近访问等状态通过本地状态持久化。
- 拖拽排序通过浏览器拖拽事件实现。

### 7.3 知识库详情与文档管理

**功能是什么**

- 上传文档。
- 导入网页链接。
- 搜索文件和筛选文件状态。
- 启用 / 禁用文档。
- 删除文档。
- 分页查看文档。
- 预览文档内容。

**怎么操作**

1. 进入某个知识库详情页。
2. 点击“上传文件”选择本地文件，也可以拖拽上传。
3. 通过“导入链接”粘贴网页地址批量导入。
4. 上传后在列表中查看文件名、分块数、上传时间和状态。
5. 如果某个文件不需要参与检索，可以切换为禁用。
6. 需要删除时直接点击删除按钮。

**用到的技术**

- 文档上传由前后端接口配合完成，支持批量和分块处理思路。
- PDF、DOCX、TXT 等文档会先被解析成文本。
- 网页导入本质上是网页正文抓取加文档入库。
- 文件列表使用表格渲染，分页由前端分页组件控制。
- 文件状态切换用于控制哪些文档参与后续检索。

### 7.4 RAG 检索与问答

**功能是什么**

- 文档向量化。
- 选择 LangChain 或原生实现。
- 切换检索策略。
- 调整 Top-K、阈值、向量权重、BM25 权重。
- 查看回答、来源和处理过程。

**怎么操作**

1. 上传完文档后先执行向量化处理。
2. 在检索框中输入问题。
3. 选择检索实现方式。
4. 根据需要调整检索参数。
5. 点击执行检索查看回答。
6. 重点查看参考来源，确认答案来自哪些文件。

**用到的技术**

- RAG 采用“先检索，后生成”的模式。
- 向量检索依赖 FAISS。
- 关键词检索通常用于补足精确匹配能力。
- 混合检索可以兼顾语义召回和关键词召回。
- `Rerank` 用于对检索结果再排序，提升答案质量。
- 流式输出让用户更快看到回答过程。

### 7.5 聊天问答

**功能是什么**

- 多轮聊天。
- 知识库增强问答。
- 模型选择。
- 深度思考开关。
- 来源溯源展开。
- 点赞、点踩、复制、重新生成。
- 停止生成。
- 图片和语音输入。

**怎么操作**

1. 进入聊天页后先选择会话。
2. 需要新聊天时点击“新对话”。
3. 选择一个模型。
4. 需要基于知识库回答时打开知识库问答。
5. 输入问题并发送。
6. 如需中断，点停止按钮。
7. 可以展开来源查看引用依据。

**用到的技术**

- 聊天消息支持 Markdown 渲染。
- 引用来源以列表方式展示，方便追溯。
- 语音输入适合快速提问。
- 深度思考属于推理过程控制，通常会带来更完整但更慢的回答。
- 会话列表用于保存上下文，保证多轮连续性。

### 7.6 任务模式

**功能是什么**

- 用自然语言描述复杂任务。
- 使用示例任务快速开始。
- 选择知识库和联网搜索。
- 观察执行步骤。
- 保存历史任务。

**怎么操作**

1. 打开任务模式页。
2. 输入任务需求或选择示例任务。
3. 按需开启知识库和联网搜索。
4. 选择执行模型。
5. 点击开始执行。

**用到的技术**

- 前端把任务执行过程拆成步骤时间线。
- 历史任务保存在前端状态里，方便重复执行。
- 可以结合知识库检索和联网搜索，增强任务结果。

## 8. 面向客户的总结口径

如果你要对客户做一句话介绍，可以这样说：

> 这个系统不是单纯的聊天机器人，而是一个把知识库、文件管理、RAG 检索、任务执行、模型管理、知识广场和协作分享整合在一起的知识平台。用户可以上传自己的资料，系统会先解析和向量化，再通过检索增强生成给出答案，同时保留来源和历史记录，保证可追溯、可管理、可扩展。

如果你要强调技术路线，可以这样说：

> 前端用 Vue3 和 TypeScript 做交互，后端用 FastAPI 提供接口，检索层使用 FAISS 和 LangChain，生成层接入 Ollama，本地数据使用 MySQL 和文件存储共同支撑，整体形成了“上传 - 解析 - 检索 - 生成 - 追溯 - 协作”的完整闭环。
