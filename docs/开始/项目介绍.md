# KnowledgeRAG-GZHU 项目介绍文档

> 适用场景：项目汇报、课程答辩、客户介绍、项目 README 前置说明。  
> 口径说明：本文档以当前项目已有页面、后端模块和仓库文档为依据，重点介绍项目定位、建设目标、核心流程、功能模块、技术架构和应用价值。

## 一、项目概述

KnowledgeRAG-GZHU，也可称为 ASF-RAG，是一个基于 RAG（Retrieval-Augmented Generation，检索增强生成）技术的智能知识管理与问答平台。

系统的核心思路是：先把用户自己的文档、资料和网页内容整理成知识库，再通过文档解析、文本切分、向量化、混合检索和大语言模型生成，让用户可以直接用自然语言向知识库提问，并得到有依据、可追溯的回答。

一句话概括：

> KnowledgeRAG-GZHU 不是单纯的聊天机器人，而是一个围绕“资料入库、智能检索、可靠问答、任务执行、协作分享”构建的知识服务平台。

## 二、项目背景

在学习、科研、办公和团队协作中，用户经常会面对大量分散资料，例如论文、教材、项目文档、制度文件、会议材料和网页信息。传统处理方式主要依赖人工阅读、关键词搜索和手动归纳，存在几个明显问题：

1. 信息分散，资料多了以后难以统一管理。
2. 搜索结果通常只返回文件或链接，用户仍然需要自己阅读和筛选。
3. 普通大模型缺少用户私有资料上下文，容易回答泛泛而谈，甚至产生幻觉。
4. 团队知识沉淀不稳定，资料共享、版本管理和协作成本较高。
5. 复杂任务往往需要多次搜索、整理和总结，单轮问答难以完成完整工作流。

本项目正是为了解决这些问题：让用户把资料变成可检索、可问答、可追溯、可协作的知识资产。

## 三、建设目标

项目建设目标可以概括为五点：

| 目标 | 说明 |
| --- | --- |
| 知识可管理 | 以知识库为单位组织文档，支持创建、上传、筛选、预览、删除和设置 |
| 问答有依据 | 回答前先检索相关资料，生成时结合原文片段，降低模型幻觉 |
| 来源可追溯 | 回答结果展示参考来源，用户可以核对答案依据 |
| 任务可执行 | 通过 Agent 任务模式处理资料整理、分析总结等复杂任务 |
| 系统可扩展 | 前后端分离，支持模型管理、接口扩展、移动端和协作分享 |

## 四、系统核心业务闭环

项目的主线不是“直接问模型”，而是围绕知识库形成一个完整闭环：

```text
创建知识库
  → 上传 PDF / DOCX / TXT 等资料
  → 文档解析与文本切分
  → 向量化并建立检索索引
  → 用户发起问题或任务
  → 系统进行向量检索 / 关键词检索 / 混合检索
  → 大语言模型结合检索结果生成回答
  → 返回答案、来源和处理过程
  → 沉淀到历史记录、文件管理或共享协作场景
```

这个闭环让系统既能管理资料，也能使用资料；既能生成答案，也能解释答案从哪里来。

## 五、主要功能模块

### 5.1 用户认证与个人中心

系统提供登录、注册、密码重置、用户信息维护等基础能力。用户登录后才能进入主应用，并可在用户中心维护头像、姓名、邮箱、简介、语言偏好和账号设置。

### 5.2 知识库管理

知识库是系统最重要的数据组织单元。用户可以新建知识库、搜索知识库、星标收藏、置顶排序、拖拽排序、查看最近访问，也可以进入详情页管理该知识库下的文档和检索配置。

### 5.3 文档上传与文件管理

系统支持将 PDF、DOCX、TXT 等资料上传到知识库，也支持导入网页链接。上传后的文档可以在知识库详情页管理，也可以在全局文件管理页集中查看、预览和删除。

### 5.4 RAG 检索与问答

RAG 是项目的核心能力。系统支持 LangChain 实现和原生实现两种检索问答方式，并提供 Top-K、相似度阈值、向量权重、BM25 权重、Rerank 重排序等配置项。用户可以在知识库详情页直接执行检索，查看最终回答、参考来源和详细处理过程。

### 5.5 多轮 AI 对话

聊天页提供更自然的问答入口。用户可以创建会话、切换模型、开启知识库问答、选择知识库、配置检索策略，并在回答中查看来源溯源。聊天过程支持流式输出、停止生成、重新生成、复制内容、点赞点踩、图片预览和语音输入等交互。

### 5.6 Agent 任务模式

任务模式面向更复杂的工作。用户可以用自然语言描述目标，系统根据任务内容进行步骤拆解，并按需结合知识库检索或联网搜索。它适合资料整理、对比分析、报告总结、问题排查等需要多步推理的场景。

### 5.7 知识图谱

项目提供知识图谱相关配置和展示能力，可以从文档中抽取实体与关系，并以图结构辅助用户理解知识之间的联系。它适合用于展示文档内容的结构化关系，也是系统区别于普通文档问答工具的重要扩展方向。

### 5.8 文档创作

文档创作页提供大纲生成、摘要生成、文本翻译、格式优化和内容扩写等能力。它把大语言模型从“回答问题”扩展到“辅助写作”，适合论文、报告、说明文和项目材料整理。

### 5.9 知识广场与协作分享

知识广场用于发现、发布和订阅共享知识库。共享详情页进一步提供文档列表、上传、预览、协作者管理、权限角色、版本历史和评论区等能力，让个人知识库可以扩展为团队知识资产。

### 5.10 模型管理与系统设置

系统提供 Ollama 模型管理页，可查看本地模型、下载模型、删除模型并配置服务地址。系统设置页集中管理 API Key、多数据源、OCR、版本管理、角色权限、RAG 评估、企业工具、多模型、合规中心、外观和系统监控等配置入口。

### 5.11 移动端应用

仓库包含 React Native + Expo 移动端项目，覆盖登录、知识库列表、知识库详情、聊天、任务模式和设置等核心页面，方便在移动设备上访问主要知识服务能力。

## 六、技术架构

项目采用前后端分离架构，整体可以分为客户端层、服务层、模型与检索层、存储层。

```text
用户
  │
  ├─ Web 前端：Vue 3 + TypeScript + Vite + TDesign
  ├─ 移动端：React Native + Expo
  │
  └─ HTTP / SSE
        │
        ▼
FastAPI 后端服务
  ├─ 用户认证与权限
  ├─ 知识库与文档管理
  ├─ 文档解析与向量化
  ├─ RAG 检索问答
  ├─ Agent 任务执行
  ├─ 知识图谱
  ├─ 模型与系统配置
  └─ 分享协作与历史记录
        │
        ├─ 模型服务：Ollama / 可扩展云端模型
        ├─ 检索组件：FAISS / BM25 / Rerank
        └─ 存储组件：MySQL / SQLite / 本地文件 / 向量索引
```

### 6.1 前端技术

前端使用 Vue 3、TypeScript、Vite、TDesign Vue Next、Pinia、Vue Router、Axios 等技术构建。页面结构清晰，覆盖知识库、聊天、任务、文件、模型、设置、广场和用户中心等功能。

### 6.2 后端技术

后端使用 FastAPI 和 Uvicorn 提供 API 与 SSE 流式接口，负责用户认证、知识库管理、文档处理、聊天会话、RAG 检索、Agent 执行、知识图谱、审计、反馈、数据源和企业配置等服务。

### 6.3 检索与生成技术

系统结合 FAISS 向量检索、BM25 关键词检索、混合检索策略和可选重排序机制，把用户问题与知识库中最相关的文档片段匹配起来，再交给大语言模型生成回答。

### 6.4 模型接入

项目重点支持本地 Ollama 模型服务，适合私有化部署和低成本演示。同时，系统设置和后端模块也为 OpenAI、DeepSeek、混元等云端模型接入保留了扩展空间。

## 七、RAG 工作原理

RAG 的核心是“先检索，后生成”。在本项目中，它主要分为两个阶段。

### 7.1 入库阶段

```text
上传文档
  → 解析文本
  → 文本切分
  → 生成 Embedding 向量
  → 写入 FAISS 向量索引
  → 保存文档元数据
```

这个阶段把普通文档变成可计算、可检索的知识片段。

### 7.2 问答阶段

```text
用户提问
  → 问题向量化
  → 检索相关文档片段
  → 混合排序与可选 Rerank
  → 构造包含上下文的 Prompt
  → LLM 生成回答
  → 返回答案与来源
```

与普通聊天相比，RAG 问答的优势是回答有知识库上下文支撑，并且可以展示来源，便于用户核对。

| 亮点 | 说明 |
| --- | --- |

| 协作分享能力 | 通过知识广场、共享详情、评论和版本等能力支持团队知识流转 |

## 九、典型使用流程

### 9.1 普通知识库问答流程

1. 用户登录系统。
2. 创建一个新的知识库。
3. 上传 PDF、DOCX、TXT 等资料，或导入网页链接。
4. 等待文档解析，并执行向量化。
5. 在知识库详情页输入问题。
6. 调整检索策略、Top-K、阈值等参数。
7. 查看系统生成的回答、来源和处理过程。

### 9.2 聊天增强流程

1. 进入 AI 对话页。
2. 新建会话并选择模型。
3. 开启知识库问答。
4. 选择参与问答的知识库。
5. 输入问题，查看流式回答和引用来源。
6. 对回答进行复制、重新生成、点赞点踩或停止生成。

### 9.3 任务执行流程

1. 进入任务模式页。
2. 输入复杂任务目标，例如“根据资料总结某个主题的研究现状”。
3. 选择是否使用知识库、是否联网搜索。
4. 点击开始执行。
5. 查看步骤流转、任务结果和历史记录。

### 9.4 协作分享流程

1. 在知识广场浏览或搜索知识库。
2. 进入共享详情页查看文档、版本和评论。
3. 根据权限进行订阅、上传、评论或协作者管理。
4. 将个人知识沉淀扩展为团队共享资产。

## 十二、项目价值

### 12.1 对普通用户

用户不再需要在大量文档中手动翻找信息，可以直接提问、总结、对比和生成内容，大幅提升资料使用效率。

### 12.2 对团队组织

团队可以把分散资料沉淀成共享知识库，通过权限、分享、评论和版本机制促进知识协作，减少重复沟通成本。

### 12.3 对开发与教学

项目覆盖前端、后端、RAG、Agent、向量检索、知识图谱、移动端和部署脚本，适合作为课程设计、毕业设计、RAG 实践和 AI 应用开发案例。

## 十三、总结口径

如果用于项目汇报，可以这样介绍：

> 本项目是一个基于 RAG 技术的智能知识管理与问答系统。它通过文档上传、解析、向量化和混合检索，把用户私有资料转化为可问答的知识库；再结合大语言模型生成有依据、可追溯的回答。系统不仅支持知识库问答，还提供多轮聊天、Agent 任务模式、文件管理、文档创作、模型管理、知识广场和协作分享等能力，形成了从知识沉淀到知识使用的完整闭环。

如果用于技术答辩，可以这样介绍：

> 系统采用 Vue 3 + TypeScript 构建前端，FastAPI 提供后端接口和 SSE 流式响应，底层结合 FAISS 向量检索、BM25 关键词检索、LangChain / 原生 RAG 流程以及 Ollama 本地模型服务，实现了“上传 - 解析 - 检索 - 生成 - 溯源 - 协作”的知识服务流程。项目兼顾可用性、可追溯性、本地部署和扩展能力，适合个人、团队和教学科研场景。
