# PPT 配套讲稿

> 口径原则：只讲当前页面已经能看到、能操作的功能，不把后台预留但未在前台展示的能力硬写进去。

---

## 第 1 页 封面

**页面标题**

基于 RAG 的智能知识管理与问答系统

**讲稿**

各位老师、各位同学大家好，我今天介绍的是我们这个基于检索增强生成技术的智能知识管理与问答系统。这个系统把知识库管理、文档处理、智能问答、任务执行、模型管理和协作分享整合到一个平台里，目标是让用户不只是“查资料”，而是能够直接“问知识库”，并且得到有依据、可追溯的答案。

---

## 第 2 页 项目背景

**页面标题**

项目背景与目标

**讲稿**

传统搜索通常只能返回一堆链接，用户还需要自己筛选信息，效率比较低。我们这个系统的思路是把资料先放进知识库，再通过检索增强生成技术把相关内容召回，最后由模型组织成自然语言回答。这样做的好处是，一方面能提高问答准确率，另一方面也能减少大模型“胡编”的情况。

---

## 第 3 页 技术架构

**页面标题**

系统技术架构

**页面内容建议**

- 前端：Vue 3 + TypeScript + Vite
- 后端：FastAPI + Uvicorn
- 检索：FAISS + LangChain
- 模型：Ollama
- 数据：MySQL + 本地文件存储
- 可视化：TDesign + Sigma.js

**讲稿**

系统采用前后端分离架构。前端使用 Vue 3 和 TypeScript 负责页面交互，后端使用 FastAPI 提供接口服务，检索层用 FAISS 做向量检索，并结合 LangChain 和 Ollama 完成问答生成。整体上形成了“上传资料、解析入库、向量检索、生成回答、返回来源”的闭环。

**页面标题**

知识库管理

**页面内容建议**

- 新建知识库

- 星标收藏
- 置顶排序

- 删除知识库

知识库页是系统最核心的入口。用户可以在这里新建知识库、搜索知识库、把常用知识库星标或置顶，也可以通过拖拽调整顺序。这个页面的作用是把不同主题的资料做成独立的知识单元，后续所有上传、检索和问答都会围绕具体知识库展开。

## 第 7 页 知识库详情页

**页面标题**

知识库详情与文档管理

**页面内容建议**

- 导入网页链接
- 文件搜索与筛选
- 启用 / 禁用
- 删除文档
- 分页浏览

**讲稿**

进入知识库详情页之后，用户可以看到这个知识库下面的所有文档。这里支持上传本地文件，也支持通过网页链接导入内容。文档列表里可以按状态筛选、搜索和分页查看，还可以直接启用或禁用某个文档，控制它是否参与后续检索。这个页面本质上就是知识库的数据入口和管理入口。

## 第 8 页 检索与问答页

**页面标题**

RAG 检索与回答

**页面内容建议**

- LangChain / 原生实现切换
- 检索策略配置
- Top-K
- 相似度阈值
- BM25 权重
- 向量权重
- Rerank

**讲稿**

知识库详情页里最关键的一块就是检索问答。当前页面支持两种实现方式切换，一个是 LangChain 版本，一个是原生实现版本，同时还可以调节检索策略、Top-K、阈值、BM25 权重和向量权重。用户先完成向量化处理，再输入问题执行检索，系统会返回最终回答、参考来源和详细处理过程，这样答案不仅能看，还能追溯。

## 第 10 页 聊天页

**页面标题**

多轮聊天问答

**页面内容建议**

- 会话列表
- 新对话
- 模型选择
- 知识库问答开关
- 检索策略配置
- 来源溯源
- 点赞 / 点踩 / 复制 / 重新生成
- 停止生成
- 语音输入

**讲稿**

聊天页是用户最常用的交互入口。这里支持多轮会话，支持模型切换，也支持打开知识库增强模式，让回答基于当前知识库生成。聊天结果可以看到引用来源，用户还能对回答进行点赞、点踩、复制和重新生成。如果回答太长或者不合适，还可以直接停止生成。这个页面体现的是系统的自然语言交互能力。

## 第 11 页 任务模式页

**页面标题**

任务模式

**页面内容建议**

- 自然语言描述任务
- 示例任务
- 知识库选择
- 联网搜索
- 执行模型选择
- 步骤展示
- 历史任务

**讲稿**

任务模式比普通聊天更进一步，它不是只回答一个问题，而是可以执行一个完整任务。用户只要用自然语言描述目标，系统就会自动拆解步骤，并且在需要时调用知识库或联网搜索。这个页面适合做资料整理、分析报告、总结归纳这类复杂任务。它体现的是系统从“问答”走向“任务助手”的能力。
